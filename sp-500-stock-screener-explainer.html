A stock screener built with ML and NLP, and how I tested it

A two-stage pipeline that ranks long and short candidates across the S&P 500. An XGBoost classifier trained on triple-barrier labels does the main ranking, and FinBERT scores live news sentiment on the shortlist. I built it on my own.

XGBoost + FinBERT · ~1.3M daily observations · triple-barrier labels · walk-forward: top-15 ~48% vs 29% base · Educational project, not investment advice

The pipeline

S&P 500 tickers are pulled from Wikipedia, priced with yfinance, and turned into technical + macro features. A single gradient-boosting model scores every stock; the strongest longs and shorts are then passed to a financial-language model for a sentiment cross-check. The output is a ranked leaderboard written to latest_signals.csv and served through a Streamlit dashboard.

Stage 1: the XGBoost classifier

target: triple-barrier label = 1 if +4% take-profit is hit before −4% stop-loss within 5 trading days // not a raw forward return
features (5 groups): momentum (RSI, MACD, lagged returns) · volatility (Bollinger, ATR) · volume (surge, VWAP dev.) · macro (SPY, QQQ, SMH, VIX, 10Y) · relative performance
one model across all tickers, not one per stock // learns market-wide patterns, not per-ticker history
regularised: max_depth 5 · subsample 0.8 · colsample_bytree 0.8
output: P(hit +4% before −4%) → rank → top 15 longs · bottom 5 shorts

The triple-barrier target is the key design choice. Instead of asking "did the price go up in 5 days", it asks "would a trade with a +4% target and a −4% stop have closed green". That reflects how a risk-managed position actually resolves. The label is computed only from future highs and lows, with no lookahead into the model's own predictions.

Stage 2: FinBERT sentiment

The top 15 and bottom 5 candidates by model confidence are passed to FinBERT (ProsusAI/finbert), a BERT model fine-tuned on financial news, analyst reports and earnings calls. Live headlines for each name are fetched and scored, and the final leaderboard combines the two signals. A bullish call needs high model confidence and positive news sentiment, not either on its own.

Validation, and what it doesn't show

  • No lookaheadThe triple-barrier label uses only future highs/lows; the dashboard adds expanding-window walk-forward testing so the model is always scored on data after its training window.
  • Overfit guardsRegularisation (shallow trees, row/column subsampling) and a single market-wide model rather than per-ticker fits.
  • SurvivorshipTraining uses current S&P 500 constituents, so delisted or removed names are absent. That flatters the historical performance.
  • FrictionsNo transaction costs, slippage or borrow fees; sentiment is headline-level only. So this is a screener and research tool, not a live-traded strategy.

Dashboard & build

The Streamlit dashboard shows the leaderboard, the walk-forward validation, and a Monte-Carlo portfolio-risk simulator (VaR / CVaR) over a chosen basket. The screener's latest signals also show up, read-only, inside my personal dashboard, purely as information.
